# HostelFresh: Full-Stack Meal Menu & Feedback System

Welcome to the HostelFresh project! This is a comprehensive solution for managing hostel mess menus and student feedback, featuring a Web dashboard and a native-ready Mobile application.

## 📁 Project Structure

- **`/client`**: React (Vite) frontend optimized for Web and Mobile (via Capacitor). Includes Bottom Navigation, Glassmorphism UI, and Haptic feedback.
- **`/server`**: Node.js/Express backend with MongoDB integration. Handles Menu scheduling, Allergy engine, and JWT Auth.
- **`/mobile`**: (Optional) Dedicated React Native (Expo) project for those who prefer a purely native UI implementation.
- **`/android`**: The native Android project generated by Capacitor for the main client.

---

## 🚀 How to Run the App

### 1. Prerequisites
- Node.js (v18+)
- MongoDB (Local or Atlas)
- Android Studio (for mobile testing)

### 2. Startup (One-Command)
From this root directory, run:
```bash
npm install
npm run dev
```
*This will start both the Backend (Port 5000) and the Web Frontend (Port 5173) simultaneously.*

### 3. Running as a Mobile App (Android)
To see the native app experience:
1.  **Configure IP**: Open `client/.env` and set `VITE_API_URL` to your computer's local IP (e.g., `http://192.168.1.5:5000`).
2.  **Navigate to Client**: `cd client`
3.  **Sync & Open**:
    ```bash
    npm run cap:sync
    npm run cap:open
    ```
    *This opens Android Studio. Simply hit "Run"!*

## ✨ Features

### Students
- **Smart Menu**: Daily/Weekly views with timing indicators.
- **Allergy Alerts**: Automated warnings if a menu item contains your restricted ingredients.
- **Interactive Feedback**: 1-5 star ratings with tactile vibration (Haptics).
- **Notifications**: Registered for Push Notifications for real-time menu updates.

### Admins
- **Menu Management**: Update food items instantly.
- **Risk Analytics**: View charts on student allergies and critical risks for the day.
- **Exportable Reports**: Generate professional PDF reports of mess popularity and risks.

## 🛠️ Setup Tips
- **DB Connection**: Configure `server/.env` with your `MONGODB_URI`.
- **Branding**: Native app icons and splash screens are already generated and located in the project assets.
- **Safe Area**: The UI is optimized for devices with notches and bottom home bars.
